Execution Context in JavaScript

📌 How JavaScript Works Internally

JavaScript is a single-threaded, synchronous language. It executes code line by line using:

  • Call Stack → Manages execution contexts
  • Memory Heap → Stores variables and objects

👉 Execution Flow:

  1. Global Execution Context (GEC) is created
  2. Creation Phase runs (memory allocation)
  3. Execution Phase runs (code executes line by line)
  4. When a function is called → new Execution Context is created
  5. Contexts are pushed/popped from Call Stack

📌 Execution Context Phases

1. Creation Phase

  • Variables → undefined
  • Functions → fully hoisted
  • Scope & this → determined

2. Execution Phase

  • Code runs line by line
  • Variables get actual values
  • Functions are executed

📌 Code Execution Sequence (IMPORTANT)

  1. Global Execution Context created
  2. All variables hoisted → undefined
  3. Functions stored in memory
  4. Execution starts line by line
  5. Function call → pushed to Call Stack
  6. Inner function → new context created
  7. Function finishes → popped from stack

📌 Scope Chain

JavaScript first looks for variables in current scope, then moves to parent scope, and finally global scope.
